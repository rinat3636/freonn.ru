## 2. Что работает хорошо

1. **Server-side мета-теги для динамических лендингов** — `/ventilyaciya-moskva`, `/kondicionirovanie-khimki`, `/ventilyaciya-sklad`, статьи блога и др. получают уникальные `<title>`, `description`, `canonical`, `og:*` и `twitter:*`.
2. **JSON-LD для услуг и городов** — `Service` + `FAQPage` внедряются на geo-страницах; `Article` — на статьях блога.
3. **301-редиректы** — `/about` → `/o-kompanii`, `/garantiya` → `/garantii`, `/shop/*` → `/uslugi` и другие legacy-URL отдают 301.
4. **robots.txt** корректно закрывает `/api/`, `/auth/`, `/admin/`, черновики и старые WordPress-пути; `Sitemap` указывает на `sitemap-index.xml`.

6. **Open Graph / Twitter Card** — полный набор базовых тегов с размером изображения 1200×630.

## 3. Критические проблемы

### 3.1. 65 внутренних страниц отдают мета-теги главной страницы

Серверная функция `getServerSeo` (файл `server/_core/vite.ts` и `server/seoInjection.ts`) не покрывает статические и list-страницы. В результате поисковый бот получает:

```html
<title>
  Freonn — инженерная компания | Монтаж вентиляции, кондиционирования в Москве
</title>
<meta
  name="description"
  content="Freonn — проектирование, монтаж и обслуживание..."
/>
<link rel="canonical" href="https://freonn.ru/" />
```

на страницах `/o-kompanii`, `/ceny`, `/blog`, `/faq`, `/obekty`, `/akcii`, `/novosti`, `/vakansii`, `/dokumenty`, `/partnery`, `/sertifikaty`, `/rekvizity`, `/garantii`, `/karta-sajta`, `/politika-konfidencialnosti`, `/slovar`, `/slovar/*`, `/kalkulyator-inzhenernyh-sistem`, `/kejs/*`, `/ceny-na-*`, `/ceny/*`, `/kommercheskie-obekty`, `/promyshlennye-obekty`, `/premium-obekty` и др.

**Почему это критично:**

- Дублирующий `<title>` и `description` на 65 URL — прямой сигнал низкого качества.
- `canonical` указывает на `/`, то есть поисковик считает эти страницы дубликатами главной и может не индексировать их.
- Клиентский `useSEO` обновляет мета-теги только после рендера JS; Яндекс и менее продвинутые краулеры могут не дождаться.

**Рекомендация:** расширить `serverSeo` и `buildPageJsonLd` в `server/seoInjection.ts`, чтобы они возвращали те же title/description/canonical/JSON-LD, что и клиентский `useSEO` для перечисленных страниц.

### 3.2. Нет server-side JSON-LD для статических/list-страниц

На 66 URL (включая главную) в HTML присутствует только `@graph` из `index.html` (`Organization`, `WebSite`). `AboutPage`, `CollectionPage`, `ContactPage`, `FAQPage`, `WebPage` для конкретных маршрутов не внедряются серверно.

**Рекомендация:** добавить `buildPageJsonLd` для `/o-kompanii`, `/ceny`, `/blog`, `/faq`, `/contacts`, `/obekty`, `/uslugi`, `/slovar`, `/kejs/:slug`, `/slovar/:slug`, `/ceny/:slug`, `/ceny-na-*` и т.д.

### 3.3. Весь контент рендерится в JS: H1 и текст отсутствуют в исходном HTML

Все 2 168 проверенных URL возвращают `<div id="root"></div>` без H1 и основного текста. Сервер подменяет только `<head>`, тело страницы пустое до выполнения React.

**Риски:**

- Google способен рендерить SPA, но Яндекс и некоторые агрегаторы могут видеть пустую страницу.
- Первичная индексация идёт по пустому `<body>`, что снижает релевантность.

**Рекомендация:** для ключевых money-страниц (`/`, `/moskva`, `/ventilyaciya`, `/ventilyaciya-moskva` и топ-20 geo) внедрить статическую генерацию / SSR / prerender (`vite-plugin-ssg` или генерацию HTML на этапе сборки). Для остальных можно оставить SPA, но с обязательным заполнением `<head>`.

## 4. Важные проблемы

### 4.1. Дубли в sitemap

- `sitemap-static.xml` содержит `/slovar` и `/kalkulyator-inzhenernyh-sistem` дважды (добавлены и через `VALID_STATIC_PATHS`, и отдельным блоком в `scripts/generate-sitemap.ts`).
- `sitemap-moscow-core.xml` пересекается с `sitemap-cities.xml`, `sitemap-services-geo.xml` и `sitemap-services-object.xml` (37 URL встречаются в нескольких файлах).
- `sitemap.xml` дублирует `sitemap-index.xml` (оба — sitemapindex).

**Рекомендация:**

- Убрать повторное добавление `/slovar` и `/kalkulyator-inzhenernyh-sistem` в `generate-sitemap.ts`.
- Либо исключить core-URL из других sub-sitemap, либо принять наличие `sitemap-moscow-core.xml` как приоритетный список, но не индексировать его отдельно в `sitemap-index.xml` одновременно с полными sub-sitemap (избыточно).
- `sitemap.xml` лучше отдавать с 301 на `sitemap-index.xml` либо удалить.

### 4.2. Open Graph type не переключается server-side

Для статей блога `og:type` в серверном HTML остаётся `website`, а не `article`. `article:published_time`, `article:author` и `article:section` отсутствуют в серверном HTML.

**Рекомендация:** расширить `injectServerSeo`, чтобы для `/blog/:slug` добавлялись `og:type=article` и `article:*` мета-теги.

### 4.3. `Organization.sameAs` содержит собственный URL

В `index.html`:

```json
"sameAs": [
  "https://freonn.ru",
  "https://vk.com/freonn",
  ...
]
```

`sameAs` предназначен для внешних профилей, не для сайта компании.

**Рекомендация:** убрать `https://freonn.ru` из `sameAs`.
